# Реализация middleware на go.
```
Клиентский запрос прежде чем попасть в обработчик роута должен пройти через “конвеерную ленту” в виде нескольких обработчиков middleware.
```
```
Middleware-обработчик может быть любым. Это может быть логгер или проверка доступа или настройка CORS. Все эти модули хочется реализовать отдельно от основных обработчиков роутов с бизнес-логикой.
```
